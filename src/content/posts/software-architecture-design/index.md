打算开一个系列总结一下关于软件学习的内容，理想情况希望能用这个系列的文章解决大多数软件设计时的问题，但实际上软件架构就是人与系统复杂度的斗争，未来软件的规模只会继续增长，遇到的工程问题一时半会看不出来会迎来完结。

第一篇打算就从最基础的部分开始写起了，正好想了一个比较有趣的开头，就从这里说起吧。（如果你想看能够立刻用到的开发范式，可以直接去看DDD的部分，PS：暂时没写到那里）

几乎所有程序都是从一个 Hello world 开始的，但好像很少有人从这里开始教一个软件的设计。实际上即使是一个在命令行显示 Hello World 的程序也具有软件的所有内容，就比如下面这段代码。

```C
int main(int argc, char*[] argv){
  char* name = "World";
  char* str = strcat("Hello ",name);
  printf("%s",str);
}
```

上面这个是一个稍稍复杂一点的 Hello World，把字符串拆开后我们其实就可以看到一个经典的三层软件架构：

- 数据层：`name` 变量，保存了名称数据；在软件里这一层一般用于和底层文件/数据库/API进行交互，用于为上层提供数据及提供数据修改的功能
- 业务层：`str` 变量，将数据根据业务需要进行加工；这一层一般对数据进行加工，或进行软件需要实际进行的业务
- 表示层：`printf`函数，将软件信息输出，表示出来，一般是使用界面等形式

一般而言在设计一个软件的时候，我们都可以根据这个简单的三层架构入手，再根据我们的需要进行修改。









但实际上架构本身只是一个"标准答案"，软件其实是一个动态运行的代码，而不是一个静态的金字塔。
